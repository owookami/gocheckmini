1. 개 요
 1.1 OpenAPI 서비스 개요 가. 배경 및 필요성
   1) 공공정보의 개방·활용에 대한 사회적 요구 확산
     - 국가자원은 공공영역을 넘어 사적 영역의 다양한 사회적, 경제적 가치 창출에 활용될 수 있는 잠재성 내재
     - 민간 영역의 자원과 융·복합을 통한 신규 비즈니스 창출이 용이한 형태의 국가 자원 개방·공유에 대한 수용 증대
     - 건축정보 중 건축물대장, 건축인허가, 주택인허가, 폐쇄말소대장 등을 개방하여 민간 활용 활성화에 기여
   2) OpenAPI기반으로 개발/제공함으로써 실시간 연계/활용이 가능하고 다른 정보와의 융합서비스 개발에 용의하게 정보 제공
  나. OpenAPI 서비스 설명
   건축서비스산업 정보체계(건축HUB) 시스템의 건축정보를 관련기관 및 민간에서 쉽게 제공받아 활용할 수 있도록 표준화 방식으로 개방한 공유서비스
 
 1.2 기대효과
    - 건축서비스산업 정보체계(건축HUB) 시스템에서 제공되는 있는 데이터를 활용하기 쉬운 OpenAPI 기반으로 일반 개발자에 공개하여 대국민의 접근성을 높이는 서비스
 
 
 
 
 
 
 
 
 
1.3 참고사항
    - OpenAPI 요청 및 응답에 나오는 시군구코드 및 법정동코드는 행정기관에서 제공받은 표준코드이며, 행정표준코드관리시스템 (https://www.code.go.kr) - 코드검색 – 법정동코드목록조회 메뉴 에서 확인 가능합니다.

다. 페이징(pagination) 방법
   1) 페이징 설명
    - API 요청시 1회 요청 가능한 목록 수(numOfRows) 최대 100건 제한
    - 100건 이상의 데이터 혹은 전체 목록이 필요할 시 페이징 기능 이용
 
   2) 페이징 방법
    - 페이징 관련 변수 설명





    - 페이징 방법
       (원하는 데이터 건 수)÷(페이지당 목록 수(numOfRows))를 계산하여 나온
       수의 소수점자리 올림 값 만큼 페이지 번호(pageNo)를 반복하여 API 요청
       * 원하는 데이터 건 수가 전체 데이터일 경우, 1회 API 요청 후 응답메세지의
         totalCount 값에서 전체 데이터 건 수를 확인 가능
